## 🏗️ Architecture Decisions

### Server vs Client Components

| Component | Type | Reason |
|-----------|------|--------|
| `page.tsx` | Server | Static composition, no state |
| `layout.tsx` | Server | Metadata, fonts, JSON-LD |
| All sections | Server | Pure rendering, no interactivity |
| `Navbar` | Client | Scroll tracking, mobile toggle |
| `FAQSection` | Client | Accordion open/close state |

This maximizes RSC (React Server Component) usage — the vast majority of HTML is rendered on the server, resulting in minimal JS hydration.

### Why No State Management Library?

The landing page needs only two tiny client-side states:
1. Active nav section (Navbar)
2. Open FAQ item (FAQSection)

Both are local `useState` — no Redux, Zustand, or Jotai needed.

### Icon System

All SVG icons are centralized in `components/ui/Icon.tsx`. This means:
- No icon library bundle weight (Lucide, Heroicons)
- Only the exact paths used in this project
- Easy to extend by adding to the `PATHS` record

### Design Tokens

Tailwind config extends with semantic tokens mirroring the original CSS variables:
```ts
brand.red       = #E8141E
neutral.dark    = #111111
status.taken    = #10B981
status.pending  = #F59E0B
```

## 🔍 SEO Implementation

### Metadata API
- Dynamic `<title>` with template via `layout.tsx`
- Full Open Graph (og:image, og:type, og:locale=en_IN)
- Twitter Card (summary_large_image)
- App links for Google Play and App Store indexing

### Structured Data (JSON-LD)
Three schemas are injected via `next/script`:
1. **Organization** — Company info, contact, social profiles
2. **SoftwareApplication** — App rating, price, OS compatibility
3. **FAQPage** — Rich results for FAQ questions in SERP

### Technical SEO
- `/sitemap.xml` — Auto-generated by `app/sitemap.ts`
- `/robots.txt` — Auto-generated by `app/robots.ts`
- Semantic HTML5: `<section>`, `<article>`, `<nav>`, `<header>`, `<footer>`, `<blockquote>`, `<cite>`, `<ol>` for steps
- Proper heading hierarchy: `h1` (hero) → `h2` (section) → `h3/h4` (cards)
- `aria-labelledby` connecting sections to their headings
- `aria-expanded` on FAQ accordion buttons
- `aria-current="page"` on active nav link
- Skip-to-content link for keyboard navigation
- All decorative SVGs have `aria-hidden="true"`
- All interactive elements have accessible labels

## ⚡ Performance Optimizations

- **`next/font`** — Poppins + Manrope loaded with `display: swap`, zero layout shift
- **Server Components** — Sections rendered as HTML, minimal client JS
- **Code splitting** — Each section is independently imported; unused sections don't ship JS
- **`prefers-reduced-motion`** — Animations disabled for users who prefer it
- **Lazy loading** — Browser-native for below-fold content
- **Security headers** — X-Content-Type-Options, X-Frame-Options set in `next.config.ts`

## 🗺️ Extending the App

### Adding a New Section
1. Create `components/sections/NewSection.tsx`
2. Add content to `data/index.ts`
3. Add types to `types/index.ts`
4. Import and render in `app/page.tsx`

### Adding a New Page (e.g. Blog)
1. Create `app/blog/page.tsx`
2. Export `metadata` from it
3. Add to `sitemap.ts`

### Adding Icons
Add the SVG path to the `PATHS` record in `components/ui/Icon.tsx`:
```ts
"my-icon": <path d="..." />
```

## 🚢 Deployment

Optimized for **Vercel** (zero-config):
```bash
vercel deploy
```

For other platforms, set:
- `output: 'standalone'` in `next.config.ts` for Docker
- `output: 'export'` for static hosting (disables dynamic routes)
